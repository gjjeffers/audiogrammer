# Security Policy

## Supported Versions

Audiogrammer is an actively developed desktop application. Security fixes are applied to the latest version on the `main` branch. We recommend always running the most recent release.

| Version | Supported          |
| ------- | ------------------ |
| latest (main) | :white_check_mark: |
| older versions | :x:           |

## Reporting a Vulnerability

If you discover a security vulnerability in Audiogrammer, please report it responsibly.

**Please do not open a public issue for security vulnerabilities.**

Instead, use one of the following private channels:

- Use GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) feature on this repository (Security tab → Report a vulnerability).
- Alternatively, contact the maintainer directly through their GitHub profile.

When reporting, please include:

- A description of the vulnerability and its potential impact.
- Steps to reproduce the issue.
- The version or commit affected.
- Any relevant logs, screenshots, or proof-of-concept details.

## What to Expect

- We will acknowledge your report as soon as possible.
- We will investigate and keep you informed of our progress.
- Once a fix is available, we will release it and credit you for the discovery if you wish.

## Scope

Audiogrammer runs fully offline and does not require API keys or network access for its core functionality. It does download Whisper model files on first use. Please keep this in mind when assessing the impact of a potential issue.

Thank you for helping keep Audiogrammer and its users safe.
